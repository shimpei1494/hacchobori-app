## Next.jsの情報収集方法
Next.js devtool mcpを利用し、最新のドキュメントやベストプラクティスを確認して、より保守性やパフォーマンス性の高い設計・コードを提案してください

## キャッシュ戦略
キャッシュに関する設計や実装を行う際は、必ず [`docs/caching-strategy.md`](docs/caching-strategy.md) を参照してください。
このプロジェクトではNext.js 16のCache Componentsを使用したシンプルで一貫性のあるキャッシュ戦略を採用しています。

重要なポイント：
- 全てのキャッシュタグは `src/lib/cache-tags.ts` の定数を使用
- Server Actionsでは `updateTag()` を直接呼び出す
- ID単位のキャッシュタグは作成しない（シンプルさ優先）
- ユーザー固有データ（セッション依存）はキャッシュしない

## コード品質とフォーマット
- Biomeによるlintエラーは、作業終了時に `npm run check:fix` を実行すれば自動修正されます
- 作業中に個別ファイルのBiome修正を実行する必要はありません
- 変更の最後にコマンド実行してください
